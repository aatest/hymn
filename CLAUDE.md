# CLAUDE.md — 찬송가·복음성가 악보 뷰어

## 프로젝트 개요

교회 예배/찬양팀을 위한 **악보 이미지 뷰어** 웹앱.
단일 HTML 파일(`hymn-viewer_20260313_2.html`)에 CSS + JS가 모두 포함된 SPA(Single Page Application)이며, 서버 없이 로컬 브라우저에서 동작한다.

## 핵심 파일

```
hymn-viewer_20260313_2.html   ← 전체 앱 (HTML + CSS + JS, ~2170줄)
```

## 기술 스택

- **프론트엔드**: Vanilla HTML/CSS/JS (프레임워크 없음)
- **폰트**: Google Fonts (Noto Serif KR, Noto Sans KR, Playfair Display)
- **저장소**: IndexedDB (`HymnViewerDB` → `songs` 스토어)
- **내보내기/불러오기**: JSON 파일

## 데이터 구조

### 내장 카탈로그 (검색용 목차)

| 변수 | 설명 | 곡 수 | 형식 |
|------|------|-------|------|
| `HYMN_DB` | 새찬송가 | 645곡 | `{n: 번호, t: 제목}` |
| `GOSPEL_DB` | 복음성가 | 487곡 | `{n: 번호, t: 제목, a: 아티스트}` |

- `HYMN_DB`: 1~645장 (새찬송가 전곡)
- `GOSPEL_DB`: 번호가 비연속적 (1~558번 중 487곡), 아티스트 필드는 현재 모두 빈 문자열

### 곡 데이터 (IndexedDB에 저장)

```javascript
{
  id: string,          // 고유 ID (timestamp + random)
  title: string,       // 곡 제목
  type: 'hymn' | 'gospel',
  number: string,      // "93장" 또는 "42번"
  images: string[],    // Base64 dataURL 배열 (악보 이미지)
  _order: number       // 목록 순서
}
```

### 번호 표기 규칙

- 찬송가: `번호 + "장"` (예: "93장")
- 복음성가: `번호 + "번"` (예: "42번")

## 주요 기능 목록

### 사이드바 (왼쪽)
- **검색**: 번호 또는 제목으로 필터링
- **필터 탭**: 전체 / 찬송가 / 복음성가
- **악보 폴더 설정**: 로컬 폴더 선택 → 파일명에서 번호 인식 → 이미지 자동 매핑
- **카탈로그 모달**: HYMN_DB / GOSPEL_DB에서 곡 검색 후 목록에 추가
- **+ 악보 추가**: 수동으로 곡 정보 + 이미지 등록
- **드래그 앤 드롭**: 곡 순서 변경
- **접기/펼치기**: 사이드바 토글 버튼

### 뷰어 (오른쪽)
- 악보 이미지 표시 (페이지 넘기기)
- 줌 인/아웃 (0.3x ~ 3x)
- 썸네일 스트립
- 곡 간 연속 이동 (이전곡/다음곡)
- 페이지 경계에서 자동으로 다음/이전 곡 이동

### 키보드 단축키
| 키 | 동작 |
|----|------|
| ← → | 페이지 이전/다음 |
| Shift + ← → | 곡 이전/다음 |
| + / - | 줌 인/아웃 |
| 0 | 줌 리셋 |

### 터치 제스처
- 좌/우 스와이프 → 페이지 이전/다음

### 저장/불러오기
- **저장**: JSON 파일로 내보내기 (이미지 Base64 포함)
- **불러오기**: JSON 파일 → 기존 목록에 추가 또는 교체

## 수정 시 주의사항

1. **GOSPEL_DB / HYMN_DB 수정**: 한 줄짜리 JSON 배열이므로 IDE에서 찾기 어려울 수 있음. `const GOSPEL_DB =` 또는 `const HYMN_DB =`로 검색
2. **번호 접미사**: 찬송가는 "장", 복음성가는 "번" — `addHymnFromCatalog()`, `renderCatalog()` 두 곳에서 분기 처리
3. **IndexedDB**: `HymnViewerDB`라는 이름으로 브라우저에 저장됨. DB 스키마 변경 시 `DB_VERSION` 증가 필요
4. **이미지 저장**: Base64 dataURL로 IndexedDB에 직접 저장. 대용량 이미지가 많으면 브라우저 저장소 한도에 주의
5. **단일 파일**: 모든 코드가 한 파일에 있으므로, 수정 시 CSS/HTML/JS 영역 구분에 유의
6. **색상 변경**: CSS 변수(`--accent` 등)와 하드코딩된 색상 모두 변경해야 일관된 테마 적용 가능
